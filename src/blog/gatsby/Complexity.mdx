## 시작
똑같은 문제를 해결하는 알고리즘이라도, 어떻게 작성되어 있는가에 따라 속도와 메모리의 차이가 있습니다. 그렇기 때문에 다양한 알고리즘 중 어느 알고리즘이 더 적합한 가를 생각해 볼 필요가 있습니다. 각 알고리즘을 비교 분석하기 위해 복잡도를 정의하고 계산합니다.
<br>
 알고리즘의 복잡도 계산 항목에는 시간 복잡도와 공간 복잡도가 있습니다.
 이것을 통해 문제를 해결하기 위해 활용한 코드가 얼마나 효율적인가 계산 해보려 합니다.
<br>
<br>
<br>

## 시간 복잡도
 특정 알고리즘이 어떤 문제를 해결하는데 걸리는 시간으로 정의합니다.
 하지만, 정확히 말하면 프로세스가 수행하는 연산을 수치화 한 것입니다.
 명령어의 실행시간은 하드웨어, 프로그래밍 언어에 따라 차이가 나기 때문에 실제 시간이 아닌 실행 횟수를 고려하는 것입니다.
<br>
<br>
<br>

## 공간 복잡도
 특정 알고리즘이 어떤 문제를 해결하는데 필요한 자원 공간의 양을 정의합니다. <br>
 총 공간 요구 = 고정 공간 요구 + 가변 공간 요구
 $$
   S(P) = c + S_p (N)
 $$
 고정 공간은 입력과 출력의 횟수나 크기에 관계없는 공간의 요구, 가변 공간은 함수가 순환 호출을 할 경우 추가되는 공간, 즉 동적으로 필요한 공간을 뜻합니다.

 좋은 알고리즘은 시간도 적게 걸리고 자원의 사용도 적은 것 이지만,
 시간과 공간은 반비례적인 경향이 있기 때문에 주로 시간 복잡도로 판단합니다.

 ```
   def func(a,n):
      a = a + 1   # 1
      for i in range(n):   # 2
         a = a + 1
 ```
 만약 이러한 함수가 있습니다. 입력값 n의 크기에 따라 동작의 수가 커진다는 것을 알 수 있습니다.
 하지만 #1 은 n의 크기와 상관없이 1번만 동작합니다. 이렇게 입력값에 크게 영향을 받지 않는 상수, 계수들을 제거하면 알고리즘의 실행시간을 계산할 수 있는데, 이것을 점근적 표기법이라고 부릅니다.
 점근적 표기법에는 3가지가 있습니다.
 1. 오메가(Big-Ω) 표기법  -  최선의 경우
 2. 세타(Big-Θ) 표기법  -  평균의 경우
 3. 빅오(Big-O) 표기법  -  최악의 경우

프로그램을 돌릴 때에는 최악의 경우를 고려하여 빅오 표기법을 주로 사용합니다.
프로그램이 항상 최선의 상황에서 돌아갈 수도 없고, 모든 입력을 고려하고 평균 수행 시간을 구하는것도 힘들기 때문입니다.
언어적으로도, "~쯤 걸린다" 보다 "~이내로 걸린다"가 신뢰도가 더 높습니다.

## BIG O
![Algorithm roadmap](./BigO.png)
http://bigocheatsheet.com
빅오로 표기된 시간복잡도입니다.

<br>

$$
   O(1)
$$

 입력 데이터의 크기에 상관없이 일정한 시간이 걸리는 알고리즘입니다.<br>
>  ex) 해쉬 테이플 접근, 스택에서의 push, pop
$$
   O(log n)
$$
입력 데이터가 커질수록 처리시간이 로그만큼 짧아지는 알고리즘입니다.<br>
>  ex) 이진탐색, 재귀(순기능)
$$
   O(n)
$$
입력 데이터에 비례해 처리시간이 증가하는 알고리즘입니다. <br>
>  ex) 선형 탐색
$$
   O(n log n)
$$
 입력 데이터가 커질수록  처리시간이 로그 배 만큼 늘어나는 알고리즘입니다.<br>
>  ex) 합볍 정렬, 퀵 정렬
$$
   O(n^2)
$$
 입력 데이터가 커질수록 처리시간이 급격하게 늘어나는 알고리즘입니다.<br>
>  ex) 이중 루프, 버블, 삽입, 선택 정렬
$$
   O(2^n)
$$
 입력 데이터가 커질수록 처리시간이 기하급수적으로 늘어나는 알고리즘입니다.<br>
>  ex) 피보나치 수열, 재귀(역기능)
$$
   O(2^n)
$$
 매우 느리지만 최선의 해결책이 이 알고리즘 밖에 없는 경우도 존재합니다.<br>
>  ex) 완전 탐색
